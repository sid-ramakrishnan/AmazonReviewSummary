    <article class="markdown-body entry-content" itemprop="text"><h1><a href="#askaway-vqa_with_modifications" aria-hidden="true" class="anchor" id="user-content-askaway-vqa_with_modifications"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>askaway-vqa_with_modifications</h1>
<p>This implementation is based on the paper: <a href="https://arxiv.org/abs/1704.03162" rel="nofollow">Show, Ask, Attend, and Answer: A Strong Baseline For Visual Question Answering</a>. The implementation is done in <a href="https://github.com/pytorch/pytorch">PyTorch</a>.</p>
<p>The reference code was taken from <a href="https://github.com/Cyanogenoid/pytorch-vqa">here</a>. Different modifications were made to the original code so that effects of these changes can be studied.</p>
<p>Please see the report for architecture and other details.</p>
<h2><a href="#running-the-model-as-it-is" aria-hidden="true" class="anchor" id="user-content-running-the-model-as-it-is"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running the model as it is</h2>
<p>The experiments were performed on both VQA 1.0 and VQA 2.0 datasets. They can be downloaded from <a href="http://www.visualqa.org/download.html" rel="nofollow">here</a>. After downloading the datasets, set the paths in config.py.</p>
<p>Execute the following commands after cloning the repository:</p>
<pre><code>python preprocess-images.py
python preprocess-vocab.py
python train.py
</code></pre>
<p>The results can be viewed in the logs directory by running:</p>
<pre><code>python view-log.py &lt;path to .pth log&gt;
</code></pre>
<h2><a href="#modifications-made-in-the-model" aria-hidden="true" class="anchor" id="user-content-modifications-made-in-the-model"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Modifications made in the model</h2>
<p>All the experiments were performed using Nvidia Tesla K80 GPU.</p>
<ul>
<li>
<p>The resnet was replaced with densenet by editing the preprocess-images.py file and the results were observed for 14 epochs. There was not much change observed in the performance and the later modifications were made with densenet. You can select either one by editing the preprocess-images.py file. Performance comparison on the VQA-1.0 dataset is given here. The runtime analysis for VQA-1.0 is given after GRU implementation.
<a href="https://github.com/animesh20/askaway-vqa_with_modifications/blob/master/performance_plots/Densenet1.0.png" target="_blank"><img src="https://github.com/animesh20/askaway-vqa_with_modifications/raw/master/performance_plots/Densenet1.0.png" alt="Performance comparison" style="max-width:100%;"></a></p>
</li>
<li>
<p>Results for VQA-2.0 dataset with Densenet and LSTM:
<a href="https://github.com/animesh20/askaway-vqa_with_modifications/blob/master/performance_plots/Densenet2.0.png" target="_blank"><img src="https://github.com/animesh20/askaway-vqa_with_modifications/raw/master/performance_plots/Densenet2.0.png" alt="Performance comparison" style="max-width:100%;"></a>
<a href="https://github.com/animesh20/askaway-vqa_with_modifications/blob/master/performance_plots/densenet2.0table.png" target="_blank"><img src="https://github.com/animesh20/askaway-vqa_with_modifications/raw/master/performance_plots/densenet2.0table.png" alt="Runtime comparison" style="max-width:100%;"></a></p>
</li>
<li>
<p>The LSTM was replaced with GRU and the bidirectional LSTM and GRU performance is also being compared. These changes can be observed in model.py file and the desired option can be selected by editing this file. Results on VQA-1.0 with densenet and GRU:</p>
</li>
</ul>
<p><a href="https://github.com/animesh20/askaway-vqa_with_modifications/blob/master/performance_plots/GRU1.0.png" target="_blank"><img src="https://github.com/animesh20/askaway-vqa_with_modifications/raw/master/performance_plots/GRU1.0.png" alt="Performance comparison" style="max-width:100%;"></a>

<ul>
<li>To study the fixation of the system on language priors, the image was turned off. The image features being obtained by the resnet and densenet architectures were replaced with a tensor of all 1s so that it can be determined whether the system actually uses the visual system or it just predicts based on the language and contextual model of the input question. This change was done by editing the data.py file. The first plot is for VQA-1.0 and second is for VQA-2.0:</li>
</ul>

<ul>
<li>The fixation on language priors was further studied by removing the image and stacked attention architecture all-together. The training time was considerably reduced here. This change was done by editing the model.py file. The first plot is for VQA-1.0 and second is for VQA-2.0:</li>
</ul>
